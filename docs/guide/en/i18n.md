# I18n

The library uses `TableTranslatorInterface::translate(string $id, array $parameters = [])` and category `mheads-yii-table` by default.

## Setup

Configure translator adapter and category source in DI:

- [examples/di/common.php](../../../examples/di/common.php)

Use a translator-aware table factory when filters should expose translated labels:

- [examples/usage/product-list-table-factory-with-i18n.php](../../../examples/usage/product-list-table-factory-with-i18n.php)

## Message IDs

| Message ID | Used in |
| --- | --- |
| `number_filter.exactly` | `Mheads\Yii\Table\Filter\NumberFilter::getSelect()` |
| `number_filter.more_than` | `Mheads\Yii\Table\Filter\NumberFilter::getSelect()` |
| `number_filter.less_than` | `Mheads\Yii\Table\Filter\NumberFilter::getSelect()` |
| `number_filter.range` | `Mheads\Yii\Table\Filter\NumberFilter::getSelect()` |
| `date_filter.exact` | `Mheads\Yii\Table\Filter\DateFilter::getSelect()` |
| `date_filter.range` | `Mheads\Yii\Table\Filter\DateFilter::getSelect()` |
| `date_filter.today` | `Mheads\Yii\Table\Filter\DateFilter::presetToday()` |
| `date_filter.yesterday` | `Mheads\Yii\Table\Filter\DateFilter::presetYesterday()` |
| `date_filter.tomorrow` | `Mheads\Yii\Table\Filter\DateFilter::presetTomorrow()` |
| `date_filter.current_week` | `Mheads\Yii\Table\Filter\DateFilter::presetCurrentWeek()` |
| `date_filter.previous_week` | `Mheads\Yii\Table\Filter\DateFilter::presetPreviousWeek()` |
| `date_filter.next_week` | `Mheads\Yii\Table\Filter\DateFilter::presetNextWeek()` |

## Custom date presets

If custom `DateFilterPreset` is added, set both:

- `messageId` as stable translation key, for example `date_filter.first_quarter`;
- `title` as human-readable fallback when no translator is configured.
